開発環境セットアップ

このページでは、ローカル開発環境について説明しています！

PHPでWebアプリを開発するには以下のものが必要になります。

  1. Webサーバ (Apache HTTP Server)
  2. データベースサーバ (MySQL, MariaDB)
  3. PHP
  4. コードエディタ (Visual Studio Code)
  5. Chrome, Edgeなどのブラウザ

これらのソフトウェア・ツール（フリーウェア）のうち、ブラウザとコードエディタ以外の、Apache、PHP、MySQL(MariaDB)をまとめてインストールしてくれる便利なパッケージソフトウェアがあります。それらが、『XAMPP』と『MAMP』です。

それぞれ、以下のようなOSに対応しています。

  • 『XAMPP』　→　Windows, Mac, Linux
  • 『MAMP』 → Windows, Mac

XAMPPとMAMPは開発環境には重要なものなのでぜひ理解を深めて下さい！

  • Visual Studio Code
  • XAMP
  • MAMP

Visual Studio Code

通常のエディタとコードエディタの違い

コードエディタは、Windowsのメモ帳、macのシンプルテキストとは違い、ソースコードの入力編集に特化したエディタです。

Visual Studio Codeについて

コードエディタには、多くの種類が存在しますが、2023年現在Visual Studio Code(VSCode)が定番となっています。無償で使いやすく、様々な拡張機能を追加することで、使い勝手を自分好みにカスタマイズできるなど、多くの点で優れています。

VSCodeのセットアップ

  1. ダウンロード

    以下のサイトからダウンロードできます。
    https://azure.microsoft.com/ja-jp/products/visual-studio-code/

  2. インストール

    インストーラーの指示に従ってください。

  3. インストールが終わったら、日本語化します。VSCodeを起動し、左の[Extensions]アイコンをクリックします。
  4. 検索ボックスから、「japan」と入力すると、日本語関連の拡張機能が一覧表示されます。ここでは、[Japanese Language Pack for Visual Studio Code]欄の[Install]ボタンをクリックします。
  5. 再起動を促すウィンドウが表示されたら、[Restart]をクリックします。
  6. 再起動が完了すると、メニューなどが日本語化されています。

ワークスペース

VSCodeには、複数のフォルダをまとめて管理できる「ワークスペース」という機能があります。ワークスペースは、VSCode上の概念であり、また作業空間であり、その作業空間に、物理フォルダを紐づけた構造となっています。

ワークスペースは単純に複数のフォルダを1つのワークスペースにで使用する目的でも使えますが、１つのフォルダを複数のプロジェクトに登録することが可能です。つまり、複数のプロフェクトで１つのフォルダを共通利用することも可能となっています。 また、VSCodeの設定、拡張機能などをワークスペースごとに登録することで、ワークスペースごとに専用の環境を構築することが可能です。

ワークスペースの作り方

  1. ソースコードなどを置くフォルダを作成します。
  2. 「ファイル」メニューから「名前を付けてワークスペースを保存」を選択し、ワークスペースを保存します。
  3. ファイル名は、「workspace.code-workspace」となっています。「workspace」は任意の名前を指定します。
  4. 「ファイル」メニューから、「フォルダをワークスペースに追加」を選択し、１．で作ったフォルダを指定します。

『XAMPP』（Windows版）

以下、XAMPPのセットアップについて解説します。

参考までに、XAMPPの公式サイトには、 「XAMPP は最も人気のある PHP 開発環境です」 「XAMPP は、完全に無償で MariaDB、PHP、および Perl を含んだ、簡単にインストールできる Apache ディストリビューションです。XAMPP オープン ソース パッケージは、インストールと利用が非常に簡単できるよう設定されています」とあります。

XAMPPのセットアップ手順

  1. インストール

    XAMPPは公式サイト、「Apache Friends」からダウンロードできます。
    https://www.apachefriends.org/jp/index.html

    インストールは、インストーラーの案内に従ってください。
    途中、Windowsのファイアーウォールの許可を求める画面が出たら、プライベートネットワークにチェックを入れたまま、[アクセスを許可する]をクリックします。

    XAMPPのコントロールパネルが表示されたら、インストールは完了です。コントロールパネルは、[x]をクリックして閉じます。

  2. PHPの初期設定
    pho.iniファイルの設定値を変更します。
    ファイルのパスは以下の通りです。

    C:\xampp\php\php.ini

    PHPを以下のように設定してください。
    [Date]
    date.timezone=Asia/Tokyo
  3. phpmyadminでパスワードを設定する
    セキュリティ上、通常はデータベースMariaDBにパスワードを設定しますが、学習環境などで設定しない場合は、以下3,4の作業は必要ありません。
    1. Xamppを起動し、[Apache]と[MySQL]を起動し、[MySQL]の横の[Admin]を選びます。
    2. [MySQL]が起動するので、[ユーザーアカウント]を選びます。
    3. [localhost]の[権限を編集]を選びます。
    4. [パスワードを変更する(Change password)]を選び、パスワードを変更します。
    5. ['root'@'localhost' のパスワードは正しく変更されました。]と表示されれば、パスワードの設定が完了です。
    6. このままでは、Xamppを再起動し、[Apache]と[MySQL]を起動し、[MySQL]の横の[Admin]を選ぶと、以下の画面が表示され、パスワードが設定されていないため、エラーが出ます。
  4. [config.inc.php]ファイルにパスワードを追記
    1. [Apache]と[MySQL]を[stop]します。
    2. [Apache」→「Config」→「phpMyAdmin(config.inc.php)」を選びます。
    3. [config.inc.php]のなかの、 $cfg['Servers'][$i]['password'] = ''; の空白になっている部分「''」にphpMyAdminで設定したパスワードを書き込みます。（例）['password'] = 'password';
    4. [Apache]と[MySQL]を起動し、[MySQL]の横の[Admin]を選ぶと、画面が表示されログインできています。
  5. コマンドプロンプトからの実行環境セットアップ（環境変数にmysqlとphpフォルダを追加）

    コマンドラインから、SQLのコマンドを実行したり、PHPのコマンドを実行できるように、環境変数Pathに、[C:\xampp\mysql\bin]と[C:\xampp\php]を追加します。

    1. [Windowsキー+Rキー]の同時押しで[ファイル名を指定して実行]ウィンドウを表示
    2. [SystemPropertiesAdvanced.exe]とコマンドを入力して、[OK]を押します。
    3. [システムのプロパティ]ウィンドウが開きます。
    4. 右下の[環境変数]をクリックします。
    5. システム環境変数の[Path]を選択し、[編集]をクリックします。
    6. [新規]をクリックし、 C:\xampp\mysql\bin を追加し、[OK]をクリックします。
    7. [新規]をクリックし、 C:\xampp\php を追加し、[OK]をクリックします。
    8. これでパスが通ったので、環境変数の設定は完了です。

『MAMP』（macOS版）

以下、MAMPのセットアップについて解説します。

「Macintosh」＋「Apache」＋「MySQL」＋「PHP」の各頭文字を取った略称です。 これらのソフトウェアをまとめてインストールできる開発環境構築ツールです。

参考までに、MAMPの公式サイトには、「MAMP は、テストや開発などの目的で、デスクトップ PC でWordPress を実行するために必要なすべてのツールを提供します。

モバイル デバイス上でプロジェクトを簡単にテストすることもできます。

MAMP は無料で提供され、簡単にインストールできます。MAMP は、システム上で既に実行されている既存の Apache インストールを侵害しません。スクリプトを開始したり構成ファイルを変更したりすることなく、Apache、Nginx、PHP、MySQL をインストールできます。さらに、MAMP が不要になった場合は、MAMP フォルダーを削除するだけで、すべてが元の状態に戻ります (つまり、MAMP は「通常の」システムを一切変更しません)。

『MAMP』のセットアップ手順

  1. インストール

    MAMPは公式サイトからダウンロードできます。
    https://www.mamp.info/en/mac/

    インストールは、インストーラーの案内に従ってください。
    途中、macOSのファイアーウォールの許可を求める画面が出たら、プライベートネットワークにチェックを入れたまま、[アクセスを許可する]をクリックします。

    MAMPのコントロールパネルが表示されたら、インストールは完了です。コントロールパネルは、[x]をクリックして閉じます。

  2. PHPの初期設定
    pho.iniファイルの設定値を変更します。
    ファイルのパスは以下の通りです。

    Applications/mamp\php\php.ini

    PHPを以下のように設定してください。
    [Date] date.timezone=Asia/Tokyo
  3. phpmyadminでパスワードを設定する
    セキュリティー上、通常はデータベースにパスワードを設定しますが、学習環境などで設定しない場合は、以下3,4の作業は必要ありません。
    1. MAMPを起動し、右上のWebStartとStartを起動し、[MySQL]の中の[phpAdmin]を選びます。
    2. [MySQL]が起動するので、[ユーザーアカウント]を選びます。
    3. [localhost]の[権限を編集]を選びます。
    4. [パスワードを変更する(Change password)]を選び、パスワードを変更します。
    5. ['root'@'localhost' のパスワードは正しく変更されました。]と表示されれば、パスワードの設定が完了です。
    6. このままで、MAMPを再起動し、右上の[Start]を起動し、[MySQL]の中の[phpAdmin]を選ぶと、以下の画面が表示され、パスワードが設定されていないため、エラーが出ます。
  4. [config.inc.php]ファイルにパスワードを追記
    1. [Start]を[stop]します。
    2. 「Apache」→[Config]→[phpMyAdmin(config.inc.php)]を選びます。
    3. [config.inc.php]のなかの、$cfg['Servers'][$i]['password'] = ''; の空白になっている部分「''」にphpMyadminで設定したパスワードを書き込みます。
      （例）['password']='password';
    4. [WebStart]と[Start]を起動し、[MySQL]の中の[phpAdmin]を選ぶと、画面が表示されログインできています。

URLに[localhost:8888]を入れ、Welcom to MAMPと表示されればセットアップ完了です！

MAMPの中の[htdocs]が公開フォルダになるので、[htdocs]の直下にフォルダを入れていきます。
